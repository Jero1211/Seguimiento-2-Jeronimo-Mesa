REPORTE DE INCIDENCIAS
HERNÁN OCAZIONEZ

Plataforma centralizada para la gestión, trazabilidad y optimización operativa del departamento de TI.

Planteamiento

Gestión de Incidencias en Hernán Ocazionez

La empresa dependía de canales informales (chats y llamadas), lo que impedía la trazabilidad. Sin un reporte centralizado, los incidentes críticos carecían de métricas de desempeño y seguimiento.

Objetivo

Centralizar el reporte de incidencias para asegurar que cada falla tenga un responsable, un tiempo de respuesta medible y una solución documentada bajo estándares ITIL.

Ciclo de Vida

Metodología de Desarrollo

01. ANÁLISIS

Planificación

Definición de requisitos funcionales y arquitectura base para el reporte de incidencias.

Prototipado

Implementación bajo el marco SCRUM para crear un MVP ágil para los usuarios.

Validación

Aplicación de DevSecOps para asegurar la integridad de la información reportada.

04. LANZAMIENTO
